Overview

The Project Manager GUI is a complete Java Swing application designed to manage software projects. It provides a user-friendly interface for creating, editing, and organizing projects, with support for external editor integration.

Project Management

Create, edit, and delete projects with comprehensive metadata including name, description, status, priority, and category.

Editor Integration

Configure and launch external editors (VS Code, IntelliJ, Sublime Text, etc.) to open projects directly from the GUI.

File System Integration

Open project folders in the system file manager and manage project locations relative to a configurable root directory.

Settings Management

Configure application settings including root directory and editor profiles through an intuitive settings dialog.

Architecture

The GUI follows a clean separation of concerns with a clear interface contract between the frontend and backend:

Presentation Layer

  • ProjectManagerGUI - Main application window
  • AddEditProjectDialog - Project creation/editing
  • SettingsDialog - Configuration management

Data Models

  • Project - Project data structure
  • EditorProfile - Editor configuration

Integration Interface

  • BackendInterface - Contract for backend implementation

Your Backend

  • Database/storage implementation
  • File system operations
  • External process management

Integration Guide

Follow these steps to integrate the GUI with your backend system:

Step 1: Copy GUI Files

Copy all Java files to your project maintaining the package structure gui.org.drbpatch.

Step 2: Implement BackendInterface

Create a class that implements BackendInterface with your data persistence and system integration logic.

Step 3: Create Main Application

Set up your main application class to initialize the backend and launch the GUI.

Step 4: Connect Event Handlers

Override the GUI's event handlers to use your backend implementation for all operations.

Troubleshooting

Common issues and solutions:

GUI doesn't show projects

Check that your getProjects() method returns data and that you've properly connected the refresh mechanism.

Buttons don't work

Ensure you've properly overridden the event handlers in ProjectManagerGUI to use your backend implementation.

Settings not saved

Implement the settings persistence methods in your backend and ensure they're called from the settings dialog.
