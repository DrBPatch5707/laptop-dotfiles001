SettingsDialog

Modal dialog for configuring application-level settings

Package: gui.org.drbpatch

Overview

The SettingsDialog class allows the user to set the projects root directory and manage editor profiles. Editor profiles define which external editors can be used to open projects. The dialog provides event listener placeholders for all actions and does not perform file I/O or persistence directly.

  • Configure projects root directory
  • Manage editor profiles (add, edit, remove)
  • Table-based editor profile management
  • Save/cancel settings

void setupEventHandlers()

Sets up all event listeners for the settings dialog's UI components. This method connects UI actions (button clicks, table selections) to their corresponding logic.

  • Browse Root Directory: Opens a directory chooser for the root directory field.
  • Add Editor Profile: Opens a dialog to add a new editor profile and updates the table.
  • Edit Editor Profile: Opens a dialog to edit the selected editor profile and updates the table.
  • Remove Editor Profile: Removes the selected editor profile from the table.
  • Save Settings: Collects all settings and triggers the save callback, then closes the dialog.
  • Cancel: Closes the dialog without saving changes.

This method is called during dialog initialization and is not intended to be called directly.

Usage Example:

private void setupEventHandlers() {
    browseRootButton.addActionListener(e -> { /* open directory chooser */ });
    addEditorButton.addActionListener(e -> { /* add editor profile */ });
    // ...other handlers
}
